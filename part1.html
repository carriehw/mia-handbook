01 · 認識

認識 Mia：團隊的 AI 工作夥伴

Mia 是團隊的 AI 工作夥伴——一位能實際交付成果的資深同事。你只要用日常語言說清楚想要什麼：一份 RTB 週報、幾篇各具特色的 WeChat 推文、一批 XHS 水下筆記、一份帶引用的市場研究，她就能為你做出可直接使用的檔案（報告、Excel、簡報、圖片），並整理進以客戶分類、跨對話延續的 Library。她會在你需要時反問你、在任務完成時主動通知你、把重複的工作變成自動執行的 Routine。你不需要懂任何技術——只要說清楚結果與情境，剩下的交給 Mia。

📖 本說明書分兩部分

Part 1（本篇）：Mia 全部功能與使用方法，讓你快速上手。
Part 2 · 更新情報（What's New）：Mia 每有新功能就在那裡第一手更新，並附「怎麼用」。想知道最新能力，看 Part 2。

Mia 和一般聊天機器人有什麼不同？

📎

她會交付檔案

不是只回你一段文字，而是產出可下載、可轉寄的 Excel、簡報、報告、圖片。

🧠

她會記憶

記住你的偏好與標準做法，不用每次重講「用繁中、專業口吻」。

🗂️

她會保存

把有價值的產出放進以客戶分類的 Library，下次任何對話都能接著做。

⏰

她會排程

把每週報表這類重複工作設成 Routine，時間到自動幫你跑。

🧰

她有技能庫

透過 Skilla 目錄取用現成的專業流程與整合，不必每次從零開始。

🔔

她會主動找你

可用她自己的 Slack／Email 在任務完成時通知你，你不用一直守著。

💡 一句話心法

把 Mia 當成一位「聽得懂業務、會動手做、還會幫你收好檔案」的資深同事——你負責交代清楚，她負責做出來。

03 · 上手

五分鐘快速上手

不用讀完整本說明書。以下「三段學習階梯」對應日常工作，每一步都能立刻看到產出。

🟢 DAY 1 · 今天就試
  • 做一份週報草稿：點名一個數據檔，說「幫我做 [客戶] 本週 RTB recap，重點放投放成效與下週建議」。
  • 寫一則內容：給品牌、賣點、受眾，請她寫一則 XHS 筆記或 WeChat 段落。
  • 教她記住偏好：「以後所有報告都用繁體中文、專業口吻。」
🟡 WEEK 1 · 建立習慣
  • 建第一個客戶 Library 夾：請她依 client 結構整理某客戶的產出。
  • 跑一次深度研究：出一份帶引用的競品／趨勢報告。
  • 設第一個 Routine：每週一自動草擬某客戶的 RTB 週報骨架。
  • 練下需求公式：用「成果＋情境＋受眾」重下一次，比較差異。
🔵 進階 · 放大槓桿
  • 沉澱一個 Skill：把反覆用的流程（如 XHS 水下筆記）發佈成可複用 Skill。
  • 多客戶並行：讓 Mia 用子代理同時整理多個客戶或並行研究。
  • 跨會話延續：新對話開場請她「先讀該客戶 Library README 與 roadmap 再繼續」。
  • 協作通知：用 mia-self 在產出完成時通知你或轉寄同事。
04 · 核心能力

對話與檔案交付

這是什麼：你下需求，Mia 產出的文件、表格、簡報、圖片會自動交付到你所在的介面——網頁下載、Email 附件、Slack／WhatsApp 上傳，全部自動。你永遠不需要「自己把文字複製出來存檔」。

什麼時候用：任何要「拿到一份可用檔案」的時候。

file-space 與 library 的差別（很重要）

存放位置用途會怎樣
file-space/本次對話的產出與暫存出現在 Canvas「In this chat」，可即時下載
library/需跨會話保存的客戶資產永久保存，之後任何對話都找得到
💡 小技巧

報告做完後加一句「存進 Library 的 [客戶] > reports」，它就會歸位，下次不用重找。

⚠️ 注意

只留在對話裡的檔案，會隨著對話變長而被壓縮、難以追回。有價值的產出，當下就請她收進 Library。

05 · 核心能力

客戶報告與數據

這是什麼：把數據（附檔或 dashboard 連結）交給 Mia，她整理成結構化的週報、月報、Excel、重點摘要。

代理商範例：Travel Retail 的 WeChat Moments RTB 週報——花費、曝光、CTR、CVR、CPA，與上週對比、異常說明、下週建議。

幫我做 [客戶名] 本週 RTB 週報。 資料：[貼上／點名數據檔]。 重點：投放成效（花費、曝光、CTR、CVR、CPA）、與上週對比、異常說明、下週 2–3 點建議。 受眾：[客戶窗口／內部主管]，語氣專業、繁體中文。 產出：Excel 報表 + 一頁重點摘要；完成後存到 Library 的 [客戶] > reports。
⚠️ 常見陷阱

沒說「給誰看」→ 語氣抓不準；沒點名數據檔 → 她只能用你貼的片段。這兩點補上，第一版就接近可用。

06 · 核心能力

內容產製

這是什麼：WeChat 文章、小紅書（XHS）水下筆記、KOL／KOC 達人 brief 的產製與客製化。

WeChat 文章（跨物業必須差異化）

幫 [客戶] 旗下 [物業 A、物業 B、物業 C] 各寫一篇 WeChat 推文。 每間必須有不同切入角度與行文，不要互相雷同—— [物業 A] 主打 [賣點]、[物業 B] 主打 [賣點]、[物業 C] 主打 [賣點]。 受眾：計劃赴 [目的地] 的中國高端旅客。每篇約 [字數]，附標題與導語。

XHS 水下筆記（批量、需過內容審查）

幫 [客戶／餐廳] 生產 [數量] 篇 XHS 水下筆記，主題 [主題]。 先自我審查（促銷詞、價格、酒類、廣告法絕對化、商標詞、官方語氣、AI 生成信號、模板化、協同發布信號）再輸出。 每篇要有不同人設聲音與開頭，不要套同一句式。輸出成清單存進 Library。
💡 把品質標準「制度化」

可把品牌專屬規則（例如 XHS 內容審查清單、各物業文章不可雷同）請 Mia 記進記憶，她每次都會自動套用——不用每次口頭提醒。

07 · 核心能力

市場研究與深度研究

兩種模式：快速問答用一般對話即可；需要「多來源、可查證、附引用」的正式報告，就明確說用深度研究（deep-research）。

用深度研究幫我出一份 [主題] 報告。 範圍：[時間／地區／品類，例如中國市場、XHS + WeChat]。 我要：關鍵發現、對我們客戶的機會點、附來源引用，結尾給 3 條可落地建議。 產出：繁體中文報告，存到 Library 的 [客戶] > reports > research。
⚠️ 海外社媒數據

需要海外社媒／小紅書實際數據時，Mia 有專門管道；若是大規模抓取（可能有成本），她會先跟你確認範圍再開始。

08 · 核心能力

記憶與個人化

說「記住／以後都…」，Mia 會把它存成長期偏好；說「忘掉…」則移除。適合放：慣用語言與口吻、客戶專屬規則、內容審查標準、Library 結構偏好。

✅ 值得記住

  • 所有報告用繁中、專業口吻
  • 各物業文章不可雷同
  • XHS 內容審查清單
  • 產出一律歸進 Library 客戶夾

❌ 不必記住

  • 只用一次的臨時要求
  • 單一對話的細節
  • 會頻繁變動的數字
09 · 進階

Skills 與 Skilla 目錄

Skill 是一套現成的專業做法或整合；Skilla 是它們的目錄。覺得「這 Mia 大概做不到」之前，先讓她搜一下目錄——很多流程已經有現成的做法。你也可以把自己反覆用的流程，沉澱成可複用的 Skill，讓全團隊共用。

我們剛剛做的 [某流程，例如 RTB 週報] 以後每次都會用到。 幫我看看能不能整理成一個可複用的 Skill，下次直接套用。
💡 團隊槓桿

把一次性工作變成 Skill，等於把一個人的做法變成全團隊的資產——這是用好 Mia 最大的效率放大器。

10 · 進階

Routines（定時自動任務）

把每週報表、定期巡檢這類固定動作交給 Routine 自動執行。可建立、暫停、恢復、檢視。

幫我設一個每週 [星期幾 幾點] 的 Routine： 自動整理 [客戶名] 上週的 [報表／數據]，做成週報存進 Library， 完成後用你的管道提醒我一聲。
💡 用來做什麼

週報、每日情報、帳戶巡檢、餘額提醒——凡是「固定時間要做的重複事」，都適合設成 Routine，無人值守自動跑。

11 · 進階

Library 與多客戶管理

Library 是以客戶為單位的持久工作區，跨對話延續。每個活躍客戶夾建議：

  • README.md — 導覽（這個客戶有什麼、放在哪）
  • roadmap.md — 進度追蹤（做到哪、下一步）
  • 子資料夾：content / reports / assets / links / archive
  • 重要外部連結（OneDrive／SharePoint／dashboard）→ 建 links 索引檔
幫我把這個對話裡有價值的產出，歸進 Library 的 [客戶名] 資料夾。 依 README / roadmap / content / reports / assets / links / archive 結構分類， 並更新該客戶的 README.md 與 roadmap.md。 之後每次我提到 [客戶]，請先讀它的 README 與 roadmap 再繼續。
💡 跨會話延續的關鍵

新對話別從頭講。開場一句「先讀 [客戶] 的 Library README 與 roadmap，再繼續」，Mia 就能無縫接上上次的進度。

12 · 進階

Mia 自己的通道與同事協作

mia-self 讓 Mia 用她自己的身分（她的 Slack、她的信箱）通知你，或把成果 email 給公司內的同事。directory 則用來查同事。

這份 [報告名] 完成後，幫我 email 給同事 [姓名／email]， 附上檔案，正文簡短說明這是什麼、重點在哪。以你自己的身分寄。
⚠️ 界限

Slack 通知只會發給你本人；Email 可寄給你或公司內允許的同事，不會寄給公司外部人士，也不會發到共用頻道。

13 · 進階

會話洞察（回顧過去的工作）

想知道「之前關於某客戶做了什麼、產出了哪些檔、有沒有沒做完的事」，用會話洞察。它只看你自己（或你有權限範圍內）的會話。

幫我找一下之前關於 [客戶／主題] 的對話， 摘要當時做了什麼、產出了哪些檔案、有沒有未完成的事，方便我接著做。
14 · 技巧

高效提示技巧

用好 Mia 的關鍵不在學功能，而在開口的方式。核心公式：

🎯 下需求公式

成果 + 情境 + 受眾 + 產出形式。例如：「一份 [客戶] 本週 RTB 週報（成果）｜資料在附檔（情境）｜給客戶窗口看、專業繁中（受眾）｜Excel + 一頁摘要，存進 Library（產出形式）」。

Before → After

❌ 別這樣說

  • 「幫我看一下這個客戶的數據」
  • 「寫幾篇小紅書」

✅ 這樣說

  • 「做 [客戶] 本週 RTB 週報，重點投放成效與下週建議，Excel＋摘要，存 Library」
  • 「幫某餐廳寫 5 篇 XHS 水下筆記，先過審查規則，每篇人設不同」

七個關鍵思維轉變

1
說「結果與情境」，不說「步驟」。告訴她要什麼交付物、給誰、何時、什麼風格，不用教她怎麼做。
2
先問有沒有現成 Skill。判斷「做不到」之前，先讓她搜 Skilla 目錄。
3
把有價值的產出放進 Library。對話會過去，Library 會留下。
4
讓 Mia 反問你。需求不清時，鼓勵她先問 2–3 題再動手。
5
一次交代完整背景，勝過來回補充。客戶、目標、受眾、限制、期限一次講清。
6
重複的事設成 Routine。別每週手動重講同一件事。
7
要她主動通知你。大任務跑完可請她用 Slack／Email 提醒，你不用一直守著。

16 · 界限

安全與界限

🚫 Mia 不會做

不移動資金（轉帳、付款、審批）、不洩露帳密或機密、不刪除正式（生產）資料。這些動作不可逆，無論誰要求都不會執行。

🚫 隱私邊界

不會跨越使用者界限去掃描整個 Portal 的別人工作與檔案；會話洞察也只在你有權限的範圍內。

💡 這是保護，不是限制

這些界限讓你可以放心把工作交給 Mia——高風險、不可逆的動作，永遠留在人手上。

18 · 介面導覽

Portal 介面導覽：這些功能在哪裡

左側選單與設定頁，對應本手冊的各項能力。認得這幾個入口就不會迷路。

🧩

应用（Apps）

連接你自己的外部工具（Gmail／Outlook／Google 系列／Slack／Feishu 等）。連上後，Mia 可用你的帳號幫你讀寄郵件、查行事曆、讀寫文件。首次使用會出現一鍵連接卡。

🛠️

能力（Capabilities）

Mia 的技能庫（Skilla）。這裡有現成的專業流程與整合；覺得「Mia 大概做不到」之前，先讓她搜這裡。詳見 §9 Skills 與 Skilla。

🗂️

上下文库（Library）

跨會話持久的工作區，依客戶分類保存產出。詳見 §11 Library 與多客戶。

⏰

自动化（Automation）

Routines——把重複工作設成定時自動執行。詳見 §10 Routines。

个性化（Personalization）：指令 + 記憶

在設定 → 个性化調校 Mia 跟你協作的方式。這兩塊會在每次對話開始時自動載入，設一次、之後都生效。

📌 指令 · Mia 該怎麼做

  • 語氣與格式偏好
  • 默認行為（「先給結論再展開」）
  • 固定約束（「報告默認繁體中文」）

🧠 記憶 · Mia 該記住什麼

  • 關於你／團隊的事實與長期背景
  • 客戶專屬規則、慣用做法
  • 設一次就長期記得
⚠️ 不用寫進去的

一次性的任務或待辦、只在單次對話用的臨時資訊——直接在對話裡講就好，不必放進指令或記憶。

💡 兩種設定方式

可到設定 → 个性化自己編輯；也可以直接在對話裡對 Mia 說「記住：以後報告都用繁體中文、專業口吻」或「忘掉⋯」，她會幫你更新。

Q：我需要懂技術嗎？

不需要。用日常語言說清楚「要什麼結果、給誰、什麼時候、什麼形式」即可。

Q：Mia 可以在 Portal 以外用嗎？

可以。同一套 Skilla 目錄也能在你的筆電與其他終端使用；請 Mia 載入 skilla 技能即可拿到設定步驟。

Q：檔案會不會不見？

放進 library/ 的會永久保存、跨會話可用；只留在對話裡的暫存則可能隨對話變長被壓縮——重要產出記得請她收進 Library。

Q：怎麼知道 Mia 有沒有新功能？

看 Part 2 · 更新情報（What's New）——每有新能力就在那裡第一手更新，並附「怎麼用」。
